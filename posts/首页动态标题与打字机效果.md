### 新功能开发计划：首页动态标题与打字机效果

#### 核心目标

我们的核心目标是对博客首页的主标题区域进行重构，使其内容完全可配置，并为副标题（名言）引入一个引人注目的“打字机”动画效果，以提升页面的动态感和个性化程度。

#### 第一步：配置中心改造 (`src/lib/config.ts`)

1. **新增配置项**: 我将在 `appConfig` 对象中，新增一个名为 `homePage` 的配置节。这个配置节将包含两个属性：
   - `title`: 一个字符串，用于定义博客的主标题（例如：“我的个人博客”）。
   - `subtitles`: 一个**字符串数组**，用于存放您希望循环展示的多条名言或副标题。

   **配置示例**:

   ```
   // src/lib/config.ts
   export const appConfig: AppConfig = {
     // ... 其他配置
     homePage: {
       title: '我的个人博客',
       subtitles: [
         '分享我的思考、技术和生活。',
         '代码改变世界，思想照亮未来。',
         '保持好奇，持续学习。'
       ],
     },
     // ... 其他配置
   };
   ```

#### 第二步：首页UI重构与动画实现

1. **数据驱动UI**: 我将修改首页组件 (`src/app/page.tsx`)，使其不再使用硬编码的文本，而是从 `appConfig.homePage` 中读取 `title` 和 `subtitles`。
2. **创建独立的动画组件**: 遵循原子设计原则，为了将复杂的动画逻辑与页面布局分离，我会创建一个全新的、可复用的**客户端组件**，例如 `TypingEffect.tsx`。
   - 这个组件将接收一个字符串数组（即我们配置的 `subtitles`）作为 prop。
   - 它将全权负责实现“打字机”的动画逻辑。

3. **实现打字机动画逻辑**: 在 `TypingEffect.tsx` 组件内部，我会实现以下循环动画流程：
   - **逐字打印**: 像打字一样，将当前名言的字符一个一个地添加到屏幕上。
   - **短暂暂停**: 在名言完整显示后，会有一个短暂的停顿，让访客可以阅读。
   - **逐字删除**: 暂停结束后，像按下删除键一样，将名言的字符一个一个地移除。
   - **切换内容**: 当名言被完全删除后，系统会自动切换到数组中的下一条名言，并开始新一轮的“打印-暂停-删除”循环。
